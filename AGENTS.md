# AGENTS

このドキュメントでは、開発支援エージェントの利用に関する方針と運用ルール、ならびに `notes/` ディレクトリで定義された補助資料との整合をまとめています。

## 目的

- チーム内で AI/エージェントを活用する際の統一的な指針を提供する。
- 利用上の注意点を共有し、セキュリティや品質を担保する。
- 既存ノートに記載された運用ルールを反映し、矛盾を防ぐ。

## エージェント利用方針

1. **補助的な役割に限定する**  
   エージェントは設計や実装の補助として使用し、最終判断は人間が行う。

2. **機密情報の扱いに注意する**  
    外部サービスへ機密情報を送信しない。必要な場合は匿名化・マスキングを行う。

3. **生成物の検証は必須**  
   エージェントが生成したコードやドキュメントは必ず人間がレビューし、正確性を確認する。

4. **ログと記録の保持**  
   重要な提案や意思決定は記録し、後から参照できる形で保管する。

## 推奨ワークフロー

1. 課題や仕様を整理して入力する。
2. エージェントの提案を受け取り、必要に応じて追加質問を行う。
3. 生成物をレビューし、必要な修正を加える。
4. 最終成果物をチームに共有し、コードレビューやテストを経て反映する。

## 注意点

- 依存関係やライセンスに関する提案は、公式ドキュメントで必ず裏付けを取る。
- ツールやライブラリの導入は、既存ポリシー（セキュリティ、ライセンス遵守など）に従う。
- 自動化された操作フローは、小規模な環境で検証してから本番環境に反映する。

## notes ディレクトリとの整合

### notes/rules_instructions.txt

- すべてのファイル編集提案は「SEARCH/REPLACE ブロック」の完全一致形式で行う。  
- 新規ファイルは空の SEARCH ブロック、REPLACE に本文を書く。  
- すべての返信は日本語で行い、クアドラプルバッククォート（````）を使用する。  
- 提案時は常に最新のファイル内容を信頼する。

### notes/acknowledged.txt / notes/rules-confirmation.md

- 上記ルールを理解・遵守していることを全員が確認済みである。  
- エージェントが生成する提案も、同じ形式とガイドラインに従うこと。

### notes/app-summary.md

- アプリ全体の構成や主要フローをまとめた基礎資料。  
- エージェントが設計・実装を支援する際は、このサマリーを常に参照し、矛盾がないか確認する。  
- 更新があれば速やかに AGENTS.md に反映し、最新の前提条件を共有する。
